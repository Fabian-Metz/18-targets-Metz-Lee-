What you will learn and why

Why you should use Targets:

  • maintain a reproducible workflow
  • save costly runtime for tasks that are already up to date
  • provide evidence that output aligns with the code and data

What you will learn in this workshop: * How to build a pipeline of custom functions using tar_script() , tar_edit()and tar_target() * How to visualize dependencies using tar_visnetwork() & tar_manifest() * How to run your pipeline using tar_make() * How to check for outdated targets using tar_outdated() * How to access targets to work with them using tar_read()and tar_load() * How to monitor data files for changes by establishing them as targets

A pipiline of functions , the targets Philosophy

A pipeline contains tasks to prepare datasets, run models, and summarize results for a business deliverable or research paper. The methods behind these tasks are user-defined R functions that live in R scripts, ideally in a folder called “R/” in the project. The tasks themselves are called “targets”, and they run the functions and return R objects. The targets package orchestrates the targets and stores the output objects to make your pipeline efficient, painless, and reproducible. Source

Start with a small pipeline.

The first version of the pipeline consists of three targets, or steps of computation. Our first targets do the following:

  1. Reproducibly track the NYC flights data file.
  2. Read in the data from the CSV file.
  3. Build a simple linear model predicting delays.
  4. Plot the model
  5. Report results

These steps are defined as custom functions in the 1-functions.Rmd

The _targets.R file

To initiating our targets, we express the computation in a special configuration file called _targets.R which has to be at the project’s root directory. This file is necessary to:

  1. Load any functions and global objects that the targets depend on.
  2. Set any options, including packages that the targets depend on.
  3. Declare a list of targets using tar_target(). The _targets.R script must always end with a list of target objects.

You would create this file from scratch using the tar_script() command.
